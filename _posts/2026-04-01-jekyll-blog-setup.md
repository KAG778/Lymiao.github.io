## 为什么选择 Jekyll？

作为研究者，我希望有一个简洁、可维护的博客系统来记录技术笔记和研究思考。Jekyll + GitHub Pages 的组合有几个明显的优势：

- **零成本托管**：GitHub Pages 免费提供静态站点托管
- **Markdown 写作**：用 Markdown 写文章，专注于内容本身
- **版本控制**：每篇文章都是 Git 里的一个文件，天然具备版本管理
- **自定义灵活**：完全掌控站点的样式和布局

## 博客架构设计

博客分为两个分类：

1. **技术**：记录编程技巧、工具使用、论文笔记等
2. **随笔**：分享研究心得、生活感悟等

### 分类系统

通过 Jekyll 的 `category` front matter 实现。每篇文章在 YAML 头部指定分类：

```yaml
---
title: "文章标题"
category: 技术
---
```

博客列表页通过 JavaScript 实现客户端分类过滤，无需额外页面。
